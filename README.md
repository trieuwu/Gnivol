# gnivol
### *“Trong một thế giới nơi yêu thương là sai lầm… bạn sẽ chọn điều gì?”*  

> **gnivol** (*đảo ngược của "loving"*) là một tựa game indie **Point & Click – Narrative Horror** nơi người chơi bước vào một **Dating Simulator bị đảo lộn**.  
> Mọi lựa chọn của bạn đều được theo dõi bởi một hệ thống vô hình.  
> Càng hành động theo **logic con người**, thế giới càng **vỡ ra**.

---

# 🌑 Introduction | Giới thiệu

**gnivol** là một trò chơi **Point-and-Click, Puzzle, Psychological Horror** mang nặng yếu tố **meta-fiction** và **narrative-driven gameplay**.

Bạn nhập vai vào nhân vật chính trong một **trò chơi giả lập hẹn hò**.  
Thoạt nhìn, mọi thứ giống như một visual novel bình thường.

Nhưng...
- Các NPC bắt đầu **hành xử kỳ lạ**
- Những lựa chọn **tử tế** lại bị coi là **phi logic**
- Các đoạn hội thoại **thay đổi sau mỗi lần lưu game**
- Và có thứ gì đó trong hệ thống **đang theo dõi bạn**

Game sử dụng một hệ thống gọi là:

> **Reality Stability (RS)**

Khi **RS giảm**, thế giới trở nên **phi lý** hơn.  
Khi **RS tăng**, game bắt đầu phá vỡ quy tắc logic, và các NPC dần **nhận ra người chơi phía sau màn hình**.

Bạn không chỉ chơi một trò chơi.
Bạn đang **bị trò chơi quan sát**.

# 🎮 Key Features | Tính năng nổi bật

## 🧠 Reality Stability (RS)

Một hệ thống gameplay trung tâm theo dõi **độ ổn định của thực tại**.

| Hành động | Kết quả |
|---|---|
| Làm theo logic của thế giới đảo ngược | RS giảm |
| Hành động theo logic con người | RS tăng |
| RS quá thấp | Thực tại sụp đổ, jumpscare |
| RS quá cao | NPC nhận ra người chơi |

Hiệu ứng khi RS thay đổi:

- Glitch hình ảnh
- Dialogue bị sửa đổi
- NPC nhớ các lựa chọn trước
- Sự kiện ẩn xuất hiện

---

## 🔄 Inverted Logic World

Thế giới trong **gnivol** hoạt động theo **logic đảo ngược**.

Ví dụ:

| Logic người chơi | Logic thế giới |
|---|---|
| Giúp đỡ người khác | Hành vi đáng ngờ |
| Giữ gìn sức khỏe | Bị xem là tội phạm |
| Cứu người | Làm hỏng trật tự |

Người chơi phải **học cách suy nghĩ như thế giới này** để sống sót.

## 🪞 Meta-fiction & Fourth Wall Break

Game liên tục **phá vỡ bức tường thứ tư**.

NPC có thể:

- đọc **tên người chơi**
- phản ứng với **file save**
- nhớ các **lần chơi trước**
- nói chuyện **trực tiếp với bạn**

Game đôi khi sẽ:

- tạo **file mới trên máy tính**
- thay đổi **menu**
- giả **crash game**
- hoặc thậm chí **viết log riêng**

# 🛠 Tech Stack

Game được phát triển bằng **Java** và hệ sinh thái **LibGDX**.

| Technology | Purpose |
|---|---|
| **Java** | Ngôn ngữ lập trình chính |
| **LibGDX** | Game framework |
| **Scene2D** | Hệ thống UI & tương tác |
| **Ashley ECS** | Entity Component System |
| **GLSL (LibGDX Shaders)** | Hiệu ứng glitch / distortion |
| **VisUI** | UI toolkit |
| **Gradle** | Build system |

Các hệ thống chính trong game:

- Dialogue Engine  
- Reality Stability Event System  
- NPC Awareness System  
- Meta Interaction Layer  
- Shader-based Glitch Rendering
